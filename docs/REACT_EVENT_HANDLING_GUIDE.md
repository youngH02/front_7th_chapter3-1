# 주니어를 위한 React 이벤트 핸들링 가이드 🎓

"왜 어떨 때는 `onClick={함수}`라고 쓰고, 어떨 때는 `onClick={() => 함수()}`라고 쓸까요?"
이 질문, React 처음 할 때 가장 많이 헷갈리는 부분이죠! 오늘 완벽하게 정리해드릴게요. 😉

---

## 1. 핵심 원칙: "함수를 넘겨라, 실행하지 말고!"

React의 `onClick`에는 **"클릭했을 때 실행할 함수"**를 줘야 합니다.
**"이미 실행된 결과"**를 주면 안 됩니다.

### ❌ 절대 하면 안 되는 것 (즉시 실행)

```jsx
// 🚨 이렇게 하면 앱이 켜지자마자 alert가 뜹니다! (무한 렌더링의 주범)
<button onClick={alert("안녕!")}>클릭</button>
```

- **이유**: `(` `)` 괄호를 붙이면 코드를 읽는 순간 **바로 실행**됩니다. 클릭하기도 전에요!

---

## 2. 상황별 올바른 사용법

### 상황 A: 인자(매개변수)가 필요 없을 때 🟢

그냥 **함수 이름만** 쏙 넣어주세요. 가장 깔끔합니다.

```jsx
const handleClick = () => {
  console.log("클릭됨!");
};

// ✅ 아주 좋아요! (함수 자체를 전달)
<button onClick={handleClick}>클릭</button>

// ⚠️ 굳이 이렇게 감쌀 필요 없어요 (동작은 하지만 코드가 길어짐)
<button onClick={() => handleClick()}>클릭</button>
```

### 상황 B: 인자(매개변수)를 넘겨야 할 때 🟡

이때는 **화살표 함수**로 한 번 감싸줘야 합니다.

```jsx
const handleGreet = (name) => {
  alert(name + "님 안녕하세요!");
};

// ❌ 안돼요! (렌더링 되자마자 실행됨)
<button onClick={handleGreet("철수")}>철수</button>

// ❌ 안돼요! (Event 객체가 name 자리에 들어감)
<button onClick={handleGreet}>철수</button>

// ✅ 정답! ("클릭하면 -> 이 함수를 실행해서 -> handleGreet를 호출해줘")
<button onClick={() => handleGreet("철수")}>철수</button>
```

---

## 3. 요약 (치트시트) 📝

| 상황          | 코드 예시                         | 설명                                      |
| :------------ | :-------------------------------- | :---------------------------------------- |
| **인자 없음** | `onClick={reset}`                 | 함수 이름만 전달 (Best! 👍)               |
| **인자 있음** | `onClick={() => setType("user")}` | 화살표 함수로 감싸기 (필수! ⭐)           |
| **즉시 실행** | `onClick={reset()}`               | **절대 금지!** (렌더링 시 바로 실행됨 😱) |

---

## 4. 심화: 왜 `onClose={handleReset}`이 더 좋을까?

```jsx
// 방법 1 (추천)
onClose={handleReset}

// 방법 2
onClose={() => handleReset()}
```

두 코드는 기능적으로는 거의 같습니다. 하지만 **방법 1**이 더 좋은 이유는:

1.  **코드가 짧고 읽기 쉽습니다.**
2.  **성능상 미세하게 유리합니다.** (방법 2는 렌더링될 때마다 새로운 익명 함수를 계속 만듭니다.)

**결론:** 인자가 필요 없다면 **방법 1**을 쓰세요! 😎
